# Internationalization (i18n) Workflow

## Overview

RULYX uses a custom JSON-based localization system via `LocalizationManager` (`Sources/Shared/Localizations/LocalizationManager.swift`). All user-facing strings are stored in 16 JSON files under `Sources/Shared/Localizations/`.

## Canonical API

There are exactly three ways to display localized text:

| API | When to use |
|-----|------------|
| `Text(loc: "key.name")` | Static text in SwiftUI `Text` views |
| `loc("key.name")` | String context — alerts, buttons, accessibility labels, parameterized strings |
| `locPlural("key_prefix", count: n)` | CLDR-aware plurals — replaces manual `_one`/`_other` selection |

### Examples

```swift
// Static text
Text(loc: "profile.block_back.action")

// Parameterized string
loc("time.minutes_ago").replacingOccurrences(of: "{n}", with: "\(minutes)")

// CLDR plural (handles _one/_few/_many/_other etc. per language)
locPlural("members_count", count: count)
    .replacingOccurrences(of: "{count}", with: "\(count)")

// Button title
Button(loc("actions.cancel"), role: .cancel) { ... }

// Accessibility
.accessibilityLabel(loc("account.switch_tab.hint"))

// Alert title
.alert(Text(loc: "list.detail.alert_title"), isPresented: ...)
```

**Never use:**
- `Text("key.name")` — bypasses JSON system, uses native lookup only
- `String(localized:)` — bypasses JSON system, ignores in-app language
- Hardcoded English strings

## Adding a New String

### Step 1: Add the key to `en.json`

```json
{
  "my.screen.title": "My Title",
  "my.screen.description": "This describes {name}."
}
```

Keys use dot-notation: `screen.component.description`.

### Step 2: Add to all 16 JSON files

```bash
python3 -c "
import json, os, glob
keys = {
    'my.screen.title': 'My Title',
    'my.screen.description': 'This describes {name}.',
}
loc_dir = 'Sources/Shared/Localizations'
for path in glob.glob(os.path.join(loc_dir, '*.json')):
    with open(path) as f:
        data = json.load(f)
    for k, v in keys.items():
        if k not in data:
            data[k] = v
    with open(path, 'w') as f:
        json.dump(data, f, indent=2, ensure_ascii=False)
"
```

**Important:** Non-English files require native translation. The script above fills English as placeholder — replace with real translations before shipping.

### Step 3: Sync to `Localizable.xcstrings`

```bash
python3 -c "
import json, os, glob
loc_dir = 'Sources/Shared/Localizations'
xcstrings_path = os.path.join(loc_dir, 'Localizable.xcstrings')

bundles = {}
for path in glob.glob(os.path.join(loc_dir, '*.json')):
    lang = os.path.basename(path).replace('.json', '')
    if lang == 'Localizable': continue
    with open(path) as f: bundles[lang] = json.load(f)

with open(xcstrings_path) as f: xc = json.load(f)
strings = xc.get('strings', {})

for key in ['my.screen.title', 'my.screen.description']:
    if key in strings: continue
    localizations = {}
    for lang, bundle in bundles.items():
        value = bundle.get(key, '')
        if value: localizations[lang] = {'stringUnit': {'state': 'translated', 'value': value}}
    if localizations:
        strings[key] = {'extractionState': 'manual', 'localizations': localizations}

xc['strings'] = strings
with open(xcstrings_path, 'w') as f: json.dump(xc, f, indent=2, ensure_ascii=False)
"
```

### Step 4: Validate

```bash
python3 scripts/validate-translations.py
```

Exit code is 0 only when all checks pass.

## Adding a New Language

1. Create `Sources/Shared/Localizations/{code}.json` with all keys from `en.json`
2. Add the language to `supportedLanguages` in `LocalizationManager.swift:17`
3. Add to `SUPPORTED_LANGUAGES` in `scripts/validate-translations.py`
4. Run validation

## CLDR Plural Rules

Languages have different plural categories. The system supports:

| Language | Categories |
|----------|-----------|
| en, de, it, nl, pt, es, tr | `one`, `other` |
| fr | `one` (n=0,1), `other` |
| ru | `one`, `few`, `many` |
| ar | `zero`, `one`, `two`, `few`, `many`, `other` |
| pl | `one`, `few`, `many` |
| ja, zh, ko, th, vi | `other` only |

### Adding plural keys

For `members_count`, add to each JSON file:

```json
{
  "members_count_one": "{count} member",
  "members_count_few": "{count} members",
  "members_count_many": "{count} members",
  "members_count_other": "{count} members"
}
```

Use `locPlural("members_count", count: n)` to look up the correct category for the current language. Falls back to `_other` if the specific category key is missing.

## RTL Support

Arabic (ar) flips layout direction to `.rightToLeft`. All directional SF Symbols (`chevron.right`, `arrow.forward`) must have `.flipsForRightToLeftLayoutDirection(true)` to mirror in RTL mode.

## Fallback Chain

1. Active language bundle → 2. English bundle (100% coverage) → 3. Raw key string

## Validation Script

`scripts/validate-translations.py` checks:

1. All keys in `en.json` exist in every language JSON file
2. No extra (orphaned) keys in non-English files
3. All keys exist in `Localizable.xcstrings`
4. Placeholder parity — `{n}` patterns match across all languages
5. `Localizable.xcstrings` values match JSON values per language

### CI Integration

The validation runs as part of the git push hook. A non-zero exit blocks the push.

```bash
python3 scripts/validate-translations.py
echo $?  # 0 = pass, 1 = fail
```
